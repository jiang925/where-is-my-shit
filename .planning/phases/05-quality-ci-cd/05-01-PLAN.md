# Phase 5 Step 1: Backend Quality Infrastructure

## Context
We need to establish a baseline for code quality and testing for the backend Python service. Currently, there are no tests or linting enforcement.

## Tasks

<action>

- pytest
- pytest-asyncio
- httpx
- ruff
</action>
</task>

<task>
<description>Configure Ruff and Pytest via pyproject.toml</description>
<action>
Create `pyproject.toml` with:
- [tool.ruff] configuration (line-length=88, target-version=py310)
- [tool.pytest.ini_options] configuration (asyncio_mode=auto, testpaths=["tests"])
</action>
</task>

<task>
<description>Create test scaffolding</description>
<action>

2. Create `tests/__init__.py`
3. Create `tests/conftest.py` with basic fixtures (e.g., event_loop if needed, or mock DB session)
</action>
</task>

<task>

<action>
Create `tests/test_core_db.py`:

</action>
</task>

<task>

</action>
</task>
