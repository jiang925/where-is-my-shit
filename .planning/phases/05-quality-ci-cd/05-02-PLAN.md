# Phase 5 Step 2: Frontend & Extension Quality Infrastructure

## Context
The frontend (React) and Extension (TypeScript) need automated quality checks. The frontend lacks a test runner, and the extension lacks a linter.

## Tasks
<task_list>
<task>
<description>Setup Frontend Testing Infrastructure</description>
<action>
In `ui/`:
1. Install dev dependencies: `vitest`, `jsdom`, `@testing-library/react`, `@testing-library/dom`
2. Update `vite.config.ts` to include `test: { environment: 'jsdom', globals: true, setupFiles: './src/setupTests.ts' }`
3. Create `ui/src/setupTests.ts` (import '@testing-library/jest-dom')
4. Add `"test": "vitest"` to `package.json` scripts
</action>
</task>

<task>
<description>Create Frontend Smoke Test</description>
<action>
Create `ui/src/App.test.tsx`:
- Render the `<App />` component
- Assert that the main title or a known element exists
</action>
</task>

<task>
<description>Setup Extension Linting</description>
<action>
In `extension/`:
1. Install `eslint`, `@typescript-eslint/parser`, `@typescript-eslint/eslint-plugin`
2. Create `.eslintrc.js` or `eslint.config.js` with basic TypeScript rules
3. Add `"lint": "eslint . --ext .ts,.tsx"` to `package.json` scripts
</action>
</task>

<task>
<description>Verify Setup</description>
<action>
1. Run `npm run test` in `ui/`
2. Run `npm run lint` in `extension/`
</action>
</task>
</task_list>
