# Phase 7, Part 2: Watcher Authentication

## Goal
Update the Python Watcher CLI to support the new JWT authentication mechanism. The watcher must read credentials from a config file, authenticate automatically, and handle token refreshes without user intervention.

## Context
Phase 6 secured the API. The watcher is a background process and cannot prompt for a password interactively every time. It needs a "set and forget" configuration.

<files>wims-watcher/src/client.py</files>
<action>
Modify `wims-watcher/src/client.py`:
1. Add `login(password)` method:
   - POST to `/api/v1/auth/token`.
   - Save `access_token` to `self.token`.
   - Persist token to `~/.wims/token` for restarts.
2. Add `_get_headers()` helper:
   - Return `{"Authorization": f"Bearer {self.token}"}`.
3. Update `ingest(data)`:
   - Use `_get_headers()`.
   - Wrap request in try/except block.
   - If 401:
     - Call `login()` with stored password.
     - Retry original request once.
     - If fails again, raise `AuthError`.
</action>
