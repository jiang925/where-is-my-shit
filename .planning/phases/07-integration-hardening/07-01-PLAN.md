# Phase 7, Part 1: Extension Authentication

## Goal
Update the Chrome Extension to support the new JWT authentication mechanism. The extension must handle 401 Unauthorized responses, prompt the user for a password, and persist the session token.

## Context
Phase 6 secured the API. Now the extension (the primary client) is broken because it doesn't send credentials. We need to implement a "login first" flow.
