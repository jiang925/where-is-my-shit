**5. src/types/message.ts:**
```typescript
export interface ExtractedMessage {
  content: string;
  role: 'user' | 'assistant';
  timestamp: string;        // ISO 8601
  conversationId: string;
  conversationTitle: string;
  platform: string;
  url: string;              // deep link or conversation URL
  fingerprint?: string;     // SHA-256 hash, set after extraction
}

// Matches server IngestRequest exactly
export interface IngestPayload {
  id?: string;
  conversation_id: string;
  platform: string;
  title: string;
  content: string;
  role: string;
  timestamp: string;
  url: string;
}
```

**7. src/lib/fingerprint.ts:**
- `generateFingerprint(conversationId, role, content): Promise<string>` using Web Crypto API SHA-256
- Normalize content: trim whitespace, collapse multiple spaces
- Hash format: `${conversationId}|${role}|${normalizedContent}`
- Return hex string

**8. src/lib/storage.ts:**
- Wrapper around chrome.storage.local for typed access
- `getSettings(): Promise<Settings>` — returns `{ captureEnabled: boolean, serverUrl: string, lastCaptureTimestamp: number }`
- `setSettings(partial: Partial<Settings>): Promise<void>`
- `getSeenFingerprints(): Promise<string[]>`
- `addSeenFingerprint(fp: string, maxSize?: number): Promise<void>` — LRU eviction at 10,000 entries (per user decision: checkpoint/fingerprint deduplication)
- `clearSeenFingerprints(): Promise<void>`
- Default settings: `captureEnabled: true`, `serverUrl: 'http://localhost:8000'`, `lastCaptureTimestamp: 0`

**9. src/lib/queue.ts (OfflineQueue):**
- Queue items stored in chrome.storage.local under key `wims_queue`
- Each item: `{ id: string, payload: IngestPayload, timestamp: number, retryCount: number }`
- `enqueue(payload: IngestPayload): Promise<void>` — adds to queue, triggers processQueue
- `processQueue(): Promise<void>` — iterates queue, sends each to server via ApiClient, removes on success, increments retryCount on failure, drops after 10 retries
- `getQueueSize(): Promise<number>` — for popup status display
- Uses `isProcessing` guard to prevent concurrent processing
- ALL state in chrome.storage.local (survives service worker termination per research pitfall #3)

**10. src/lib/api.ts (ApiClient):**
- `constructor(serverUrl: string)`
- `sendMessage(payload: IngestPayload): Promise<boolean>` — POST to `${serverUrl}/api/v1/ingest` with JSON body
- 10-second fetch timeout using AbortController (per research: service worker can terminate after 30s)
- Returns true on 2xx, throws on network error or non-2xx
- `checkHealth(): Promise<boolean>` — GET `${serverUrl}/api/v1/health`, returns true/false

**11. src/content/extractors/base.ts (BaseExtractor abstract class):**
```typescript
export abstract class BaseExtractor {
  abstract platform: string;
  abstract matches(): boolean;
  abstract extractMessages(): ExtractedMessage[];
  abstract getConversationTitle(): string;
  abstract getConversationId(): string;

  protected extractTextContent(element: Element): string {
    // Clone element to avoid mutating DOM
    const clone = element.cloneNode(true) as Element;

    // Replace images with reference text per user decision
    clone.querySelectorAll('img').forEach(img => {
      const alt = img.getAttribute('alt') || 'unnamed';
      const textNode = document.createTextNode(`[Image: ${alt}]`);
      img.parentNode?.replaceChild(textNode, img);
    });

    // Replace file attachments with references
    // (platform-specific selectors handled in subclasses)

    return clone.textContent?.trim() || '';
  }
}
```
