Complete WIMS Capture extension with ChatGPT, Gemini, and Perplexity extractors, offline queue, and minimal popup UI.

  <how-to-verify>
**Setup:**
1. Start WIMS server: `cd /home/pter/code/where-is-my-shit && python -m uvicorn src.app.main:app --host 0.0.0.0 --port 8000`
2. Build extension: `cd /home/pter/code/where-is-my-shit/extension && npm run build`
3. Load extension in Chrome: Navigate to `chrome://extensions`, enable Developer Mode, click "Load unpacked", select `extension/dist/` directory

**Test 1: Extension installs cleanly**
- Verify extension appears in extensions list without errors
- Click extension icon — popup should show on/off toggle (default: on), server status (green if server running), last capture time

**Test 2: ChatGPT capture (WEB-01)**
- Open https://chatgpt.com and start or open an existing conversation (/c/ URL)
- Type a message and get a response
- Check WIMS server: `curl -s -X POST http://localhost:8000/api/v1/search -H "Content-Type: application/json" -d '{"query": "your message text", "platform": "chatgpt"}' | python -m json.tool`
- Verify messages appear with correct role labels (user/assistant)

**Test 3: Gemini capture (WEB-02)**
- Open https://gemini.google.com and have a conversation
- Search WIMS for the conversation content
- Verify messages captured with platform "gemini"

**Test 4: Perplexity capture (WEB-03)**
- Open https://www.perplexity.ai and ask a question
- Search WIMS for the query/answer content
- Verify messages captured with platform "perplexity"

**Test 5: Real-time detection (WEB-04)**
- On any platform, send a new message
- Within 10 seconds, search WIMS for that message content
- Verify it appears in results

**Test 6: Toggle capture**
- Click popup, turn capture OFF
- Send a new message on ChatGPT
- Search WIMS — new message should NOT appear
- Turn capture back ON, send another message — should appear

**Test 7: Offline queue**
- Stop WIMS server (Ctrl+C)
- Send a message on ChatGPT
- Popup should show queued message count > 0
- Restart WIMS server
- Wait up to 1 minute (alarm interval)
- Search WIMS — queued message should now appear

**Test 8: Options page**
- Right-click extension icon > Options
- Verify server URL is shown (default http://localhost:8000)
- Click "Test Connection" — should show success
- Change URL to invalid, test — should show failure
