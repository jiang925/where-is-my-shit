**1. Service Worker (`extension/src/background/service-worker.ts`):**

Implement the background service worker that:
- Listens for `MESSAGES_CAPTURED` messages from content scripts
- For each message in payload, converts ExtractedMessage to IngestPayload and enqueues via OfflineQueue
- Listens for `GET_STATUS` messages from popup, responds with `{ queueSize, serverOnline, lastCaptureTimestamp }` by checking queue size and pinging health endpoint
- Listens for `TOGGLE_CAPTURE` messages from popup, updates `captureEnabled` in storage
- Sets up `chrome.alarms` for periodic queue processing (every 1 minute)
- On alarm, calls `queue.processQueue()`
- On service worker startup, immediately processes any pending queue items
- Logs `[WIMS] Service worker initialized` on startup

Message types:
```typescript
type MessageType = 'MESSAGES_CAPTURED' | 'GET_STATUS' | 'TOGGLE_CAPTURE';
```

For MESSAGES_CAPTURED: Extract conversationTitle from payload, map each ExtractedMessage to IngestPayload:
```typescript
{
  conversation_id: msg.conversationId,
  platform: msg.platform,
  title: msg.conversationTitle,
  content: msg.content,
  role: msg.role,
  timestamp: msg.timestamp,
  url: msg.url
}
```
Update `lastCaptureTimestamp` in storage after successful enqueue.

**2. Content Script Entry (`extension/src/content/index.ts`):**

Main orchestrator that:
- On load, checks `captureEnabled` from storage; if false, exits early
- Instantiates all extractors: `[new ChatGPTExtractor()]` (Gemini and Perplexity added in Plan 03)
- Finds matching extractor via `extractor.matches()`
- If no match, logs and exits
- Creates MessageObserver, starts observing with the matched extractor
- Callback on new messages: sends `chrome.runtime.sendMessage({ type: 'MESSAGES_CAPTURED', payload: { messages, conversationTitle } })`
- Adds scroll event listener (debounced 500ms) that calls `observer.triggerCheck()` — per user decision: capture on scroll for lazy-loaded history
- Listens for storage changes on `captureEnabled` to start/stop observer dynamically
- Cleans up observer on `beforeunload`

**3. MutationObserver (`extension/src/content/observers/mutation-observer.ts`):**

Implements `MessageObserver` class:
- `start(extractor: BaseExtractor, onNewMessages: callback): void`
  - Creates MutationObserver with `{ childList: true, subtree: true }` (NO attributes, NO characterData — per research pitfall #2)
  - Observes `document.querySelector('main') || document.body`
  - On mutation: calls `debouncedProcess()` (1 second debounce)
  - Does initial extraction immediately on start
- `triggerCheck(): void` — public method called on scroll, triggers debounced process
- `stop(): void` — disconnects observer, clears timers
- Private `processMessages()`:
  - Calls `extractor.extractMessages()`
  - For each message, generates fingerprint via `generateFingerprint()`
  - Checks fingerprint against in-memory Set AND persisted seen fingerprints (from storage)
  - New messages (unseen fingerprint) get added to seen set and persisted via `addSeenFingerprint()`
  - Calls callback with array of new messages only
- Maintains in-memory `seenFingerprints: Set<string>` loaded from storage on init for fast dedup checks

<task type="auto">
  <name>Task 2: ChatGPT extractor implementation</name>
  <files>extension/src/content/extractors/chatgpt.ts</files>
  <action>
Implement `ChatGPTExtractor` extending `BaseExtractor`.

**Platform detection:**
- `matches()`: `window.location.hostname === 'chatgpt.com'` (per user decision: standard chat only)
- Additionally check pathname starts with `/c/` OR is just `/` (main page with active conversation)

**Conversation identification (Claude's discretion: use platform URL IDs):**
- `getConversationId()`: Extract from URL pathname `/c/{uuid}`. If no UUID in URL, return empty string (skip capture on non-conversation pages).

**Conversation title:**
- `getConversationTitle()`: Try multiple selectors in order:
  1. `document.querySelector('h1')?.textContent` (ChatGPT shows conversation title as h1)
  2. `document.querySelector('[data-testid="conversation-title"]')?.textContent`
  3. `document.querySelector('nav a.bg-token-sidebar-surface-secondary')?.textContent` (active sidebar item)
  4. Fallback: `'Untitled Conversation'`

**Message extraction:**
- `extractMessages()`: Returns `ExtractedMessage[]`
- Query message containers. ChatGPT uses `[data-message-id]` attributes on message elements. Try these selectors in order:
  1. `document.querySelectorAll('[data-message-id]')` (primary)
  2. `document.querySelectorAll('article')` (fallback if data attributes removed)
  3. `document.querySelectorAll('[class*="agent-turn"], [class*="human-turn"]')` (last resort)
- For each message element:
  - **Role**: Check `[data-message-author-role]` attribute. Map "user" -> "user", everything else -> "assistant". If no attribute, infer from DOM position or class names.
  - **Content**: Use `extractTextContent()` from base class. Additionally:
    - Handle code blocks: Keep code inline with markdown-style backtick wrapping. Preserve language label if available from `<code class="language-*">` or code block header.
    - Handle non-text: Replace `<img>` with `[Image: alt]`, replace file attachments with `[File: name]` per user decision.
  - **Timestamp**: ChatGPT doesn't show per-message timestamps in DOM. Use `new Date().toISOString()` for capture time.
  - **URL**: `window.location.href` (deep link is the conversation URL; ChatGPT doesn't support message-level deep links reliably, so conversation URL is the best-effort per user decision)
  - **conversationId**: from `getConversationId()`
  - **conversationTitle**: from `getConversationTitle()`
  - **platform**: `'chatgpt'`

**Robustness:**
- Wrap all DOM queries in try/catch — if selector fails, log warning and return empty array (never crash the page)
- Log `[WIMS ChatGPT] Extracted N messages` on each extraction
- If no message elements found with any selector, log `[WIMS ChatGPT] No message elements found — DOM may have changed`
