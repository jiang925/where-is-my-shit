<task type="auto">
  <name>Task 1: Gemini extractor</name>
  <files>extension/src/content/extractors/gemini.ts</files>
  <action>
Implement `GeminiExtractor` extending `BaseExtractor`. Follow the exact same pattern as ChatGPTExtractor.

**Platform detection:**
- `matches()`: `window.location.hostname === 'gemini.google.com'`
- Gemini conversations live at `gemini.google.com/app/{conversation_id}` or just `gemini.google.com/app`

**Conversation identification:**
- `getConversationId()`: Extract from URL path `/app/{id}`. If no ID segment, generate one from page URL hash or use a timestamp-based fallback.

**Conversation title:**
- Try selectors in order:
  1. `document.querySelector('[data-conversation-title]')?.textContent`
  2. `document.querySelector('h1.conversation-title')?.textContent`
  3. `document.querySelector('.chat-window header')?.textContent`
  4. Active item in sidebar navigation
  5. Fallback: `'Untitled Gemini Conversation'`

**Message extraction:**
- Gemini renders messages in turn-based containers. Try selectors:
  1. `document.querySelectorAll('.conversation-container .message-content')` or similar turn containers
  2. `document.querySelectorAll('[data-message-id]')` (if Gemini uses data attributes)
  3. `document.querySelectorAll('.model-response-text, .user-query-text')` or role-based class names
  4. `document.querySelectorAll('message-content')` (web component style)
- **Role detection**: Gemini typically labels model responses distinctly from user queries. Check parent element classes or attributes for role indication. Map to 'user' or 'assistant'.
- **Content**: Use `extractTextContent()` from base. Handle code blocks and images same as ChatGPT.
- **URL**: `window.location.href`
- **Platform**: `'gemini'`

**IMPORTANT NOTE ON DOM SELECTORS:**
Gemini's DOM structure must be reverse-engineered at implementation time. The selectors above are best guesses based on common patterns. The implementor MUST:
1. Open gemini.google.com in Chrome DevTools
2. Inspect the actual DOM structure of a conversation
3. Identify the correct selectors for message containers, roles, and titles
4. Update the extractor with working selectors
5. Implement multiple fallback selectors for robustness

Log `[WIMS Gemini] No message elements found` if all selectors fail. Never crash.

**1. Perplexity Extractor (`perplexity.ts`):**

Implement `PerplexityExtractor` extending `BaseExtractor`.

**Platform detection:**
- `matches()`: `window.location.hostname === 'www.perplexity.ai'`
- Perplexity threads live at `www.perplexity.ai/search/{id}` or similar URL patterns

**Conversation identification:**
- `getConversationId()`: Extract from URL path. Perplexity uses `/search/{query-slug}-{id}` or `/thread/{id}` patterns.

**Conversation title:**
- Try selectors:
  1. `document.querySelector('h1')?.textContent` (Perplexity shows query as title)
  2. `document.querySelector('[class*="Title"]')?.textContent`
  3. First user query text as fallback title
  4. Fallback: `'Untitled Perplexity Thread'`

**Message extraction:**
- Perplexity shows Q&A pairs. Try selectors:
  1. Query blocks and answer blocks as separate containers
  2. `document.querySelectorAll('[class*="Query"], [class*="Answer"]')` or similar
  3. Perplexity uses a prose/answer section with citations
- **Role**: User queries -> 'user', Perplexity answers -> 'assistant'
- **Content**: Use `extractTextContent()`. Perplexity includes citations — capture them as inline text (e.g., "[1] source.com"), don't strip them.
- **URL**: `window.location.href`
- **Platform**: `'perplexity'`

**Same DOM discovery requirement as Gemini** — implementor must inspect real Perplexity DOM and adjust selectors. Log warnings on failure, never crash.
