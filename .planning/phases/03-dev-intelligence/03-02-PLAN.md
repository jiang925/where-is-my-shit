# Plan 03-02: Multi-Source Watcher & Robustness

**Goal:** Refactor `wims-watcher` to support multiple log sources (Claude, Antigravity, Cursor) and improve ingestion reliability.

**Rationale:** Phase 3 verification identified gaps: hardcoded Claude support and fragile "all-or-nothing" cursor updates. We need a flexible architecture to ingest from multiple dev tools.

## 1. Architecture Refactor
- **Parser Registry:** Split `parser.py` into a module `src/parsers/` with a strategy pattern.
- **Config Driven:** Move from single ENV var to a configuration file (`config.json`) defining sources and parsers.
- **Robust Ingestion:** Update cursor logic to handle partial failures (at-least-once delivery with granular checkpoints).
