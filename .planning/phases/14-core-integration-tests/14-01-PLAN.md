Step 4 - Perform search and verify results:
- Set up response promise BEFORE typing (SearchBar has 300ms debounce, fires automatically):
  ```typescript
  const searchResponse = page.waitForResponse(
    res => res.url().includes('/api/v1/search') && res.status() === 200
  );
  ```
- Fill search input: `await page.getByPlaceholder('Search your history...').fill('Kubernetes Docker')`
- Await the search response: `const response = await searchResponse`

- Wait for result content to appear in UI: `await expect(page.getByText('Kubernetes')).toBeVisible({ timeout: 10000 })`

Step 5 - Validate result card content:
- Assert that result card shows the ingested content (partial match is fine since content may be truncated)
- Assert platform metadata is displayed if visible in card

**Important implementation notes:**
- The SearchBar component uses a 300ms debounce (`useEffect` with `setTimeout`). There is NO submit button or Enter key handler. Typing triggers search automatically after the debounce delay.
- Use `page.waitForResponse` set up BEFORE the `fill()` call to catch the debounced API request.
- Do NOT use `waitForTimeout`. Use `toBeVisible()` with explicit timeout if needed.
- The `authenticatedPage` fixture sets up page.route for API auth headers, but for the ingest setup step, use the `request` context with explicit headers instead.
- Use `{ timeout: 10000 }` on result assertions to account for embedding generation time on first run.
