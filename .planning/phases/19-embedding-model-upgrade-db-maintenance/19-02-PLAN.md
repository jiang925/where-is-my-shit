  <action>
    **Create `src/app/db/compaction.py`:**

    `CompactionManager` class with these specifics:

    1. **Constructor** `__init__(self, write_threshold: int = 100)`:
       - `self.write_threshold = write_threshold` (100 writes as default — reasonable for single-user tool per research)
       - `self._write_count = 0`
       - `self._counter_lock = threading.Lock()` — protects write counter
       - `self._compaction_lock = threading.Lock()` — prevents concurrent compaction
       - `self._stop_event = threading.Event()` — signals thread to stop
       - `self._thread: threading.Thread | None = None`
       - `self._table = None` — set via `set_table()`
       - structlog logger at module level

    2. **`set_table(self, table)`**: Store reference to LanceDB table. Called during app startup after init_db().

    3. **`record_write(self, count: int = 1)`**: Thread-safe increment of write counter. If threshold exceeded after increment, spawn compaction via `_trigger_compact()`.

    4. **`_trigger_compact(self)`**: Reset counter to 0 (under counter lock), then call `_compact()` in a new daemon thread (don't block the calling API thread). Use `threading.Thread(target=self._compact, daemon=True).start()` for threshold-triggered compaction.

    5. **`_compact(self)`**:
       - Try `self._compaction_lock.acquire(blocking=False)` — if fails, log "compaction_skipped" reason="already_running" and return (skip-silently approach per user decision)
       - Log "compaction_started"
       - Count fragments before: `len(self._table.list_fragments())`
       - Call `self._table.compact_files()`
       - Count fragments after
       - Log "compaction_complete" with fragments_merged=before-after
       - Wrap in try/except, log errors
       - Release lock in finally block

    6. **`start(self)`**: Create and start a daemon thread that:
       - Runs `_compact()` once on startup (user decision: compact on startup)
       - Then enters a loop: `self._stop_event.wait(10)` — check every 10 seconds
       - Break if stop_event is set
       - This thread handles startup compaction; threshold compaction is triggered by record_write

    7. **`stop(self)`**: Set stop event, join thread with 5s timeout

    8. **Module-level singleton**: `compaction_manager = CompactionManager()`

    **Create `tests/test_compaction.py`:**
    - Test `record_write` increments counter correctly
    - Test `_should_compact` returns True after threshold writes (test the counter/threshold logic)
    - Test `_compact` skips when lock already held (mock table, acquire lock manually, call _compact, verify log)
    - Test `_compact` calls table.compact_files() and logs fragment counts (mock table)
    - Use `unittest.mock.MagicMock` for the table object
    - Use `unittest.mock.patch` for structlog assertions if needed
