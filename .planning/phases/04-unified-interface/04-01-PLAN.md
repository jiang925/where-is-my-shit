# Plan 04-01: Backend Pagination & Frontend Scaffold

We need to prepare the backend to serve the UI and handle large result sets, then set up the frontend project structure.

## Context
Currently, the `/search` endpoint returns a fixed number of results (default 10) with no way to page through them. For a search UI, we need infinite scroll, which requires `offset` support.
Additionally, `main.py` is set up to serve a simple `test.html`. We need to reconfigure it to serve a built React SPA.

<task>
<id>3</id>
<description>Initialize React Frontend project</description>
<file>ui/</file>
<instructions>
1. Create a generic `ui` directory in the project root if it doesn't exist.
2. Initialize a Vite + React + TypeScript project inside `ui/`.
   - Use `npm create vite@latest . -- --template react-ts` inside `ui/` directory.
   - Run `npm install`.
3. Install dependencies:
   - `npm install -D tailwindcss postcss autoprefixer`
   - `npx tailwindcss init -p`
   - `npm install lucide-react clsx tailwind-merge`
4. Configure `tailwind.config.js` to scan `./src/**/*.{js,ts,jsx,tsx}`.
5. Add `@tailwind base; @tailwind components; @tailwind utilities;` to `src/index.css`.
</instructions>
</task>

<task>
<id>4</id>
<description>Configure Vite build output for FastAPI integration</description>
<file>ui/vite.config.ts</file>
<instructions>
Configure Vite to build directly into the backend's static assets folder.
1. Set `build.outDir` to `'../src/static'`.
2. Set `build.emptyOutDir` to `true`.
3. Ensure the base path is relative or root-compatible (`base: '/'`).
</instructions>
</task>
