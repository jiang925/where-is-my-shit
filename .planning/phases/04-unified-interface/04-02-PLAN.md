# Plan 04-02: Search Interface Implementation

Now that the foundation is laid, we will build the actual user interface for searching and navigating history.

## Context
We need a clean, responsive interface that allows users to type a query and see results from their local vector database. We will use TanStack Query (React Query) to handle the async state, debouncing, and infinite scrolling.

## Tasks

<task>
<id>1</id>
<description>Install Frontend Logic Dependencies</description>
<file>ui/package.json</file>
<instructions>
Install libraries required for state and interaction:
`npm install @tanstack/react-query axios` (or stick to fetch if preferred, but axios is easier for params).
`npm install @radix-ui/react-slot` (common for Shadcn-like components).
</instructions>
</task>

<task>
<id>2</id>
<description>Create API Client and Hooks</description>
<file>ui/src/lib/api.ts</file>
<instructions>
1. Create a typed `search` function that calls `POST /search`.
2. Implement `useSearch` custom hook using `useInfiniteQuery`.
   - Key: `['search', query]`
   - Fetcher: calls `search` API with `offset` based on `pageParam`.
   - `getNextPageParam`: calculates next offset (current + limit).
</instructions>
</task>

<task>
<id>3</id>
<description>Build SearchBar Component</description>
<file>ui/src/components/SearchBar.tsx</file>
<instructions>
Create a centered input component.
- Visual: Large, distinct, "Google-like" or "Spotlight-like" simplicity.
- Behavior: Update local state immediately, but update parent "query" state with a debounce (300ms).
</instructions>
</task>

<task>
<id>4</id>
<description>Build ResultCard Component</description>
<file>ui/src/components/ResultCard.tsx</file>
<instructions>
Display a single search result.
- Header: Source Icon (Chrome/Claude/Terminal) + Timestamp (formatted relative, e.g., "2 hours ago").
- Body: Text snippet (limit lines/height).
- Footer: "Open" button that links to the `url` (deep link).
- Metadata: Show score/relevance on hover (optional).
</instructions>
</task>

<task>
<id>5</id>
<description>Implement Main App Layout & Infinite Scroll</description>
<file>ui/src/App.tsx</file>
<instructions>
Assemble the application.
1. Setup `QueryClientProvider`.
2. Layout: Sticky Header (SearchBar) + Main Content (Grid).
3. Grid: Map over `data.pages` from `useInfiniteQuery`.
4. Infinite Scroll: Add a "Load More" intersection observer element at the bottom (or a simple "Load More" button for V1 stability).
5. Zero State: "Start typing to search..."
6. Empty State: "No results found."
</instructions>
</task>

<task>
<id>6</id>
<description>Build and Final Verification</description>
<file>ui/</file>
<instructions>
1. Run `npm run build`.
2. Serve via backend.
3. Perform a test search.
4. Verify results appear.
5. Verify clicking a result opens the correct deep link.
</instructions>
</task>

## Verification
- [ ] User can type in search bar and see results update (debounced).
- [ ] Results display correct source icon (Web vs Dev).
- [ ] Infinite scroll (or load more) fetches next page of results.
- [ ] Clicking a result opens the target URL.
- [ ] Empty state handles "no results" gracefully.
